Userdefined Procedure

A userdefined procedure is a command sequence, that can be invoked multiple times. That way you can avoid to copy and paste command sequences you need multiple times. EOS differs between methods and procedures. Methods can access all variables, but procedurs can only access local variables and parameters. As a consequence the automatically generated window is not available within procedures. Figures created within a procedure have to be added to a window or group by hand or they stay invisible.

If you define more than one procedures they must differ in either the name or the number of parameters.

Syntax

Procedure without parameters

procedure name
  sequence
*procedure

Procedure with parameters

procedure name( parameters )
  sequence
*procedure

Procedure with parameter and return value

procedure name( parameter ):type
  sequence
  result = ...
*procedure

Parameters

Parameters are written like

name : type, name : type , ... , name : type

If you have multiple parameters with the same type, you can use the short form.

name, name, ... , name : type

Usage

In order to invoke a procedure you write

name( arguments )

Example

procedure move(r:Rectangle, angle:real)
  r.move( 100 *cos(grad),  100 *sin(angle))
  r.rotate(angle)
*procedure
i:Integer
repeat  36  times
  r:Rectangle
  move(r,  10 *i)
  i = i +  1
*repeat
