Comments

Comments are texts that are part of the source code, but not part of the program. That means they will not be executed. Every programming language has some way to add comments. They can contain anything (few exceptions) without changing the behaviour of the program.

They are needed in order to add explaining text and remarks to a program. This is necessary in order to better understand programs. You should allways write a comment, if the source code is not self explaining, or there are informations one has to know to understand the code, which are not obvious.

That way comments on the one hand help understanding a program faster and better. On the other hand if a program get more complex, it can happen that you forget yourself what you did and the comment reminds you what the program is doing.

Syntax

Line comment

// The rest of the line is a comment no matter what it contains.

Line comments are useful for adding comments behind commands.

Area comment

/* Text between /* and */ is a comment */

or as alternativ

{ text between curly braces is a comment}

Area comments can be place anywhere, where a blanc is legal. Therfore they can be placed within lines.

Example

c:Circle //That's a circle
{this is a multiline comment.
second line
third line.}
c.move(-1/*to the left*/,1/*up*/)
